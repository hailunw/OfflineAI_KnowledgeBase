标题: 基本设计阶段的成果物都有哪些？编写者是谁？
内容: 
    1. 基本设计书（概要设计书）
        内容：系统整体架构、模块划分、层级结构、整体流程、技术选型、运行环境构成。
        编写者：SE（主担当）、架构师、PL/PM
        依据：要件定義書、业务流程、客户要求、公司技术标准与架构规范。
    2. 数据库基本设计书
        内容：ER 図、テーブル一覧、主キー・外部キー、インデックス方針、データ量見積、データベース運用方針。
        编写者：DBA、データベース設計 SE
        依据：要件定義のデータ項目、业务数据规则、基本设计的模块与流程、公司 DB 设计规范。
    3. インタフェース設計書（接口设计书）
        内容：外部・内部接口一覧、呼び出し形式、パラメータ（入力 / 出力）、エラーコード、フォーマット、連携タイミング。
        编写者：SE、基盤担当者、外部連携担当者
        依据：要件定義の外部連携要求、基本设计架构、模块间依赖、外部系统提供的接口仕様。
    4. 画面設計書（画面设计书）
        内容：各画面レイアウト、項目一覧、入力チェックルール、画面遷移、権限、操作仕様。
        编写者：SE、フロントエンド担当者、UI 設計者
        依据：要件定義書、画面遷移図、业务流程、基本设计的模块与权限方針。
    5. 業務ロジック設計書（业务逻辑设计书）
        内容：核心业务规则、处理フロー、分岐条件、例外処理、計算ロジック。
        编写者：各モジュール SE、業務担当 SE
        依据：要件定義書、业务流程图、基本设计方針、客户业务规则。
    6. 共通機能設計書（共通机能设计书）
        内容：認証・認可、ログ出力、エラーハンドリング、キャッシュ、共通部品、バッチ処理方針。
        编写者：基盤 SE、アーキテクト
        依据：基本设计架构、非機能要件（性能・信頼性）、公司共通部品仕様・開発規約。
    7. テスト方針書（测试方针书）
        内容：試験レベル（単体・結合・総合）、試験範囲、環境、品質基準、重点確認箇所。
        编写者：試験リーダー、試験担当 SE
        依据：要件定義書、基本设计全体方針、品質目標、公司試験標準。
    8. 運用・保守設計書（运维设计书）
        内容：デプロイ構成、バックアップ、モニタリング、障害復旧、スケール方針、運用フロー。
        编写者：インフラ SE、運用担当者、アーキテクト
        依据：基本设计架构、非機能要件、インフラ環境、公司運用標準。
