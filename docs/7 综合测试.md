标题: 综合测试是什么？谁执行综合测试？
内容：综合测试（総合試験 / 系统测试）是在结合测试完成后，将整个系统部署在模拟生产环境中，从最终用户视角验证系统整体功能、性能、易用性、安全性、兼容性等是否满足需求定义与业务目标的测试，是系统上线前最接近真实使用场景的测试。执行者：专业测试团队（测试负责人 + 测试工程师），部分项目会邀请客户业务担当、SE 共同参与验证。

标题: 综合测试的执行者是谁？
内容：由独立的测试部门/测试小组执行；客户方业务人员参与用户验收测试（UAT），开发、SE 配合问题排查。

标题: 综合测试的测试用例是谁写的？
内容：由测试设计工程师、测试负责人主导编写；业务人员提供业务场景并协助复核，SE 补充技术相关校验点。

标题: 综合测试的测试用例是基于什么写的？
内容：
    要件定義書（需求规格书）：核心依据，覆盖所有功能、非功能需求
    业务流程图、画面迁移图：完整业务场景与操作流程
    基本设计书、详细设计书：系统整体逻辑与实现规则
    接口设计书：内外系统联动、数据交互规则
    非功能需求：性能、并发、安全、兼容性、可靠性要求
    实际业务操作手册与用户使用场景

标题: 综合测试的输入文档和成果物都有哪些？
内容：
    输入文档
        要件定義書、业务流程图、画面原型
        基本设计书、详细设计书、接口设计书
        结合测试合格报告、单体测试报告
        综合测试方针、测试设计书
        测试环境部署文档、用户操作手册
    成果物
        综合测试用例
        综合测试结果报告书
        缺陷票（不具合票）及修复验证记录
        性能 / 压力 / 兼容性测试报告
        综合测试合格报告
        用户验收测试（UAT）确认书

标题: 综合测试的测试数据是如何准备的？
内容：
    按真实全业务流程构造数据
    覆盖从前端操作→后端处理→数据存储→报表输出的完整链路，模拟用户真实操作顺序。
    使用贴近生产的业务数据
    数据格式、字段长度、业务规则、关联关系与生产一致，可抽取生产脱敏数据使用。
    覆盖全场景测试数据
    正常业务流程、分支流程、异常操作、边界值、大批量数据、并发操作数据。
    准备初始化与环境数据
    执行 SQL 脚本初始化基础数据（字典表、权限表、参数表），保证测试环境干净可重复。
    多系统联动测试数据
    涉及外部系统对接时，准备联动所需的接口数据、文件数据、中间表数据，保证端到端流转。
    性能 / 压力测试专用数据
    构造大批量数据（万级及以上），验证系统处理效率、并发能力、数据容量上限。
