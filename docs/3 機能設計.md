标题: 机能设计定義是什么？
内容: 
    相对于要件定义关注系统做什么， 
    基本设计关注有哪些模块，
    详细设计关注代码如何实现， 
    机能设计关注每个功能如何处理。
    本知识库文档，包含4个主要机能。
        1️⃣ 文档加载机能
        2️⃣ 知识库构建机能
        3️⃣ RAG检索机能
        4️⃣ 问答生成机能


标题: 文档加载机能是什么？
内容: 
    机能ID：F01
    机能名称：Markdown文档加载
    机能概要： 系统从指定目录读取 Markdown 文档， 并加载为 Document 对象。
    输入:
        项目	    内容
        目录路径	docs 目录
    处理流程:
        1 扫描 docs 目录
        2 查找所有 .md 文件
        3 使用 TextLoader 读取文件
        4 转换为 Document 对象
    输出:
    项目	            内容
    DocumentList	Markdown文档对象
    对应代码:
        DirectoryLoader
        TextLoader

标题: 知识库构建机能是什么？
内容: 
    机能ID：F02
    机能名称：向量数据库构建
    机能概要：系统将 Markdown 文档转换为向量，并构建 FAISS 向量数据库。
    输入:
        项目	        内容
        Document	Markdown内容
    处理流程
        1 MarkdownSplitter切分文档
        2 SentenceTransformer生成向量
        3 创建FAISS索引
        4 保存index.bin
        5 保存metadata.json

    流程图：    
        Markdown
           ↓
        split
           ↓
        embedding
           ↓
        FAISS
           ↓
        保存索引
    输出：
        项目	            内容
        index.bin	    FAISS索引
        metadata.json	chunk内容
    对应代码：
        init_rag_db()
        MarkdownSplitter
        SentenceTransformer
        faiss.IndexFlatIP
标题: RAG检索机能是什么？
内容: 
    机能ID：F03
    机能名称： RAG知识检索
    输入：
    项目	    内容
    用户问题	string
    处理流程：
        1 Query Rewrite
        2 生成向量
        3 FAISS搜索
        4 返回最相似chunk
    
    流程：    
        Query
         ↓
        Rewrite
         ↓
        Embedding
         ↓
        FAISS search
         ↓
        Result
    输出：
        项目	    内容
        知识内容	chunk
    对应代码：
        build_query_prompt()
        text_2_vector()
        rag_retrieval()

标题: 回答生成机能是什么？
内容: 
    机能ID： F04
    机能名称: 回答生成
    输入:
        项目	内容
        问题	Query
        知识	RAG结果
    处理流程:
        1 构建prompt
        2 调用LLM
        3 生成回答
    输出:
    项目	    内容
    Answer	文本回答
    对应代码:
        build_answer_prompt()
        llm_model.create_completion()

标题: 机能设计阶段的成果物都有哪些？编写者是谁？
内容: 
    1. 机能设计书（機能設計書）
        内容：各机能的处理流程、算法、分支 / 异常逻辑、调用关系、内部数据结构
        编写者：各模块担当 SE、开发工程师
        依据：基本设计书、业务逻辑设计书、接口设计书
    2. 程序设计书（プログラム設計書）
       内容：クラス構成、メソッド定義、パラメータ、処理フロー、呼出関係
       编写者：开发工程师、模块 SE
       依据：机能设计书、基本设计、接口仕様
    3. 数据库详细设计书（DB 詳細設計書）
       内容：テーブル最終定義、項目型・長さ、制約、インデックス、SQL 設計
       编写者：DBA、データ設計 SE、開発 SE
       依据：DB 基本设计书、机能处理逻辑
    4. 详细接口设计书（インタフェース詳細設計書）
       内容：電文形式、項目詳細、エラーコード、フォーマット、ログ出力ルール
       编写者：連携担当 SE、基盤 SE
       依据：基本設計のインタフェース仕様、外部システムインタフェース情報
    5. 画面详细设计书（画面詳細設計書）
       内容：項目制御、入力チェック詳細、イベント処理、エラーメッセージ
       编写者：フロントエンド SE、画面担当 SE
       依据：画面基本设计、业务流程、机能要求
    6. 共通部品详细设计书（共通部品詳細設計書）
       内容：共通クラス、ライブラリ、メソッド仕様、利用方法
       编写者：基盤 SE、アーキテクト
       依据：共通機能基本設計、開発標準・規約
    7. 单体测试设计书（単体試験設計書）
       内容：試験ケース、試験データ、期待値、合否判定基準
       编写者：開発 SE（自分で設計・実施）または試験担当者
       依据：机能设计书、程序设计书、业务逻辑
