标题: Agile开发是什么？
内容: 
   Agile 开发（敏捷开发） 是一种以人为核心、迭代、循序渐进的软件开发方法论，核心是快速响应变化、持续交付可用软件、高效协作。
   一、核心思想
   个体与互动 重于流程与工具
   可用的软件 重于详尽的文档
   客户协作 重于合同谈判
   响应变化 重于遵循计划
   二、关键特点
   迭代开发：把项目拆成多个短周期（迭代，通常 2～4 周），每轮交付可运行的软件。
   持续交付：小步快跑、频繁发布，尽早获得反馈并快速调整。
   拥抱变化：需求变更被视为正常且有价值，允许在开发中灵活调整。
   高频沟通：团队面对面协作，客户 / 产品方深度参与，减少信息偏差。
   三、常见实践框架
   Scrum：最主流，用 Sprint、Scrum 团队、每日站会、待办列表等管理流程。
   Kanban（看板）：可视化任务流转，限制在制品，追求流程流畅。
   XP（极限编程）：强调工程实践，如测试驱动开发、持续集成、结对编程等。
   四、和传统瀑布模式对比
   瀑布：需求→设计→开发→测试→上线，一步走完再下一步，后期改造成本极高。
   敏捷：边做边调、持续迭代、风险分散、更适配互联网 / 快速变化的业务。
   五、一句话总结
   敏捷不是一套固定流程，而是一套 “快速做、快速改、持续交付价值” 的开发理念。

标题: 你在里面的角色是什么？是如何参与Agile开发的？
内容: 
Agile开发通过短周期迭代（Sprint）逐步完成系统开发，每个迭代周期都会交付一个可以运行的系统功能。 在AI离线知识库项目中，各角色通过每日沟通和阶段性交付逐步完成系统构建。

项目主要角色：
Product Owner（产品负责人）
负责定义需求和功能优先级。

Scrum Master（流程协调者）
负责组织Agile流程并协调团队沟通。

Development Team（开发团队）
负责系统设计和代码开发。

QA / Tester（测试人员）
负责功能验证和测试。

DevOps
负责运行环境和系统部署。

--------------------------------

Daily Standup（每日站会）
团队每天进行一次15分钟的站会，每位成员回答三个问题：
1 昨天完成了什么工作  
2 今天准备做什么工作  
3 当前遇到了什么问题  

例如：

开发人员A
昨天完成了 Markdown 文档读取代码，
今天准备实现 Markdown 解析模块，
目前问题是部分 Markdown 编码解析异常。

开发人员B
昨天完成了 Embedding 模块测试，
今天准备实现 FAISS 向量数据库，
目前没有阻碍。

Scrum Master记录问题并协调解决。

完成内容：

main.py  
TextLoader

实现能力：系统可以扫描本地目录并读取 Markdown 文档。

目标：实现知识库内容解析。

完成内容：

- 将 Markdown 解析为 Question / Answer

对应代码模块：MarkdownSplitter  

解析规则：
标题 → question  
内容 → answer  

实现能力：系统可以将 Markdown 知识库转换为 QA 数据结构。

text_2_vector()

--------------------------------

metadata 包含：
question  
answer

Sprint Review（迭代评审）
每次迭代结束后，团队向 Product Owner 演示新功能，

例如：Sprint 4 Review
演示内容：系统可以自动建立 FAISS 向量数据库。

--------------------------------

Sprint Retrospective（回顾会议）

团队讨论：
哪些做得好  
哪些需要改进  

例如：
改进点：
- Markdown解析规则需要更稳定
- 向量检索需要增加相似度阈值
